# Product Truth

## What RateGuard Is

- Middleware-first API traffic control for existing apps.
- Self-hosted control plane plus realtime event spine.
- LLM token budgets and traditional API rate limiting in one product path.
- Gateway mode is optional, not the product center.

## What RateGuard Is Not

- Not a forced gateway migration.
- Not a billing-first SaaS wrapper.
- Not a plan-tier marketing site with technical garnish.

## Wedge

RateGuard wins on:

- in-process middleware for Go, Node, and Python
- self-hosted control plane and runtime contracts
- LLM plus REST traffic control in one developer path
- OSS-friendly deployment for teams that refuse proxy migration

## Copy Rules

- Use `preset` or `policy preset`, never `plan` or `billing plan`.
- Use `guardrail`, never `billing limit`.
- Use `handle`, not `username`, in API payloads.
- Dashboard stats expose `preset` only.
- Auth payloads expose `handle` and `preset` only.
